Baseline

A Network For Tennis Players

Baseline is the one-stop hub for all tennis players to take their game to the next level. With a social network experience specially tailored for tennis players, Baseline helps you play more of what you love.

Professional Design

Baseline was designed to be unique, yet familiar for the Apple ecosystem using Apple’s Design Resources. Two major iterations were completed, with the first being exploratory, and the second a deep refinement. And as the implementation continues, so do improvements to the design.

Fully Prototyped in Sketch

Baseline is fully prototyped in Sketch for early user testing and to provide a clear goal for the code implementation. The prototype is high fidelity, with great attention to detail that maps one-to-one with the final app.

Built Using The Latest & Greatest

SwiftUI is the best choice for a modern iOS app that not only allowed ease of development, but also clarity with the readability of code. My goal was to find the path of least resistance, and the SwiftUI framework proved to be the perfect choice.

A Backend Made to Scale Fast

Firebase proved to be another great choice as it provided well documented libraries for the modern serverless programming paradigm. It allowed for rapid testing of database queries from the Baseline client to populate with content. Firebase was carefully chosen for its seamless integration with Swift, and cost-effective scaling and toolset.

Organized Using Agile Methods

Through sprints and backlogs, Baseline was developed with documentation and accountability in mind. Clear goals were set and met using agile methods, down to each commit push having meaningful comments to document changes.

Full Advantage of the Environment

Diving deeper into the decision for Baseline’s tech stack, by using the tools outlined above, development was sped up considerably through features like Swift Previews. Xcode features many advantages over other IDE’s, but research and intentional code structure is needed to use them. I put in the effort to learn how to get the most out of the IDE, and it paid off through convenience and faster identification of bugs.

Code Written For Readability

One of my guiding principles when writing code is to make it simple and understandable. I always kept in mind that someone else might read my code, and I needed to write it with that intention in mind. All variables, structures, and logic flows in Baseline were written to make sense to other team members.

… And Modularity

All of my backend logic is separated from the frontend, with the only interactions being through coupling of separate classes and structs. This was done to draw a clear line between the frontend and backend, but also to allow for the easy replacement of Firebase libraries if ever needed. This approach also encapsulates data and variables retrieved from the database, allowing for easy distinguishing between database-derived variables and local variables.
